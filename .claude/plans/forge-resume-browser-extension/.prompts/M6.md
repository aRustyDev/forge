## Architecture constraints
- Overlay is shadow DOM injected by content script — prevents host CSS leakage
- Answer bank stores in Forge DB via `answer_bank` table (not chrome.storage) — "extension is thin client"
- Background worker must NOT import from `src/plugin/plugins/*` or `@forge/core/src/parser` (shared chunk constraint)
- `enrichWithParser()` at `packages/extension/src/lib/enrich-extraction.ts` already provides ParsedJobDescription in content script context — overlay reads from this
- Forge API changes needed: `answer_bank` table + `GET/PUT /api/profile/answers` endpoints
- Workday app questions extend M4's dropdown/radio filling via `FieldKind` → answer bank mapping
- Version bump to 0.1.6 on completion
