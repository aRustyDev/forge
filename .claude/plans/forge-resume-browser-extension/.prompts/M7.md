## Architecture constraints
- Config stored in Forge DB (`extension_config` table), fetched on startup via `GET /api/extension/config`
- Graceful fallback to `chrome.storage.local` when API unreachable
- Extension logs stored in Forge DB (`extension_logs` table), gated by config flag
- WebUI pages for config editor + log viewer
- `devMode` defaults to `false` in production config
- Version bump to 1.0.0 marks MVP complete
