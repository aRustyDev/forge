# T90.4: Organization Industry Combobox + Education Validation (Deferred)

> **Status:** Deferred from Phase 90 (2026-04-05). Backend (Phase 90 T90.1-T90.3, T90.5) is complete on main. This is a UI-only followup.

**Duration:** Short-Medium (3-4 tasks)

## Goal

Replace the free-text `industry` input on the organization editor with a combobox that lists existing industries from the new `industries` table, supports the "select existing or create new" pattern, and enforces the rule that education orgs (tagged `university` or `school`) cannot have an industry. Also round-trip `industry_id` through the organization repository and routes so saves persist correctly.

## Non-Goals

- Moving away from the deprecated `organizations.industry` text column (kept for backward compat)
- Bulk industry renaming / merging
- Industry autocomplete with fuzzy matching (strict name match is sufficient)

## Context

Phase 90's backend added the `industries` entity table, `industry_id` FK on organizations, CRUD APIs, and SDK resource. The UI still has a free-text `<input type="text">` for industry at `packages/webui/src/routes/data/organizations/+page.svelte` (around lines 572-575, gated by `{#if formOrgType !== 'education'}`).

Two gaps:
1. **Hide logic uses `formOrgType` instead of org tags.** Phase 90's requirement is to hide the field when the org has tags `university` or `school` (tag-based, since `org_type` is deprecated in favor of multi-tag).
2. **Save payload uses free-text `industry` only.** Need to translate the combobox selection (name or new-name input) to `industry_id` via `forge.industries.getByName()` or inline create, and include it in the organization create/update payload.

The `organization-repository.ts` create/update also needs to accept `industry_id` — it currently only persists the free-text `industry` column. This wasn't caught during Phase 90 because no existing route/test exercised the field.

## Tasks

### T90.4.1: IndustryCombobox Component

**File:** `packages/webui/src/lib/components/IndustryCombobox.svelte`

**API:**
```typescript
let {
  value,        // string | null — currently selected industry name
  industryId,   // string | null — bound FK (two-way)
  disabled,     // boolean
  placeholder,  // string
  onchange,     // (industry: Industry | null) => void
}: Props = $props()
```

**Behavior:**
1. On mount: `forge.industries.list()` → populate dropdown
2. User types → filter existing matches (case-insensitive)
3. Exact match → select that industry's `id`
4. No match + user presses Enter or blurs with non-empty value → call `forge.industries.create({ name })`, use new id
5. Clear button clears both `value` and `industryId`
6. Show "+ New: <typed>" option when input doesn't match existing

**Pattern reference:** `packages/webui/src/lib/components/OrgCombobox.svelte` — follow its structure for combobox UX, keyboard handling (arrow keys, enter, escape), and ARIA attributes.

**Acceptance Criteria:**
- [ ] Component renders with loaded industries
- [ ] Typing filters the dropdown
- [ ] Selecting an existing entry fires `onchange` with the Industry object
- [ ] "Create new" path creates industry via API and fires `onchange` with the new record
- [ ] Clear button resets state
- [ ] Keyboard navigation works (arrow keys, enter, escape)
- [ ] ARIA combobox role + announcements
